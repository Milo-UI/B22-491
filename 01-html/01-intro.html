HTML & CSS - HTML 01 - Intro

Cos'è l'HTML?

HyperText Markup Language

È un linguaggio di markup e non di programmazione. Si tratta quindi di un sistema di annotazioni utilizzato per definire la struttura e il layout di un documento. Markup infatti è un termine che si riferisce a un insieme di tag o annotazioni inseriti in un testo per specificare come questo debba essere visualizzato o interpretato.

L'HTML viene usato per strutturare il contenuto di una pagina web (immagini, testo, form, ecc).
Per farlo utilizziamo quelli che si chiamano tag HTML.

I tag rappresentano differenti tipi di contenuto o informazioni all'interno di una pagina web. Per esempio, avremo un tag apposito per le immagini, uno per i form, uno per i paragrafi di testo e così via. I tag vengono anche chiamati elementi HTML.

Questo è un tag H1, che rappresenta il titolo principale di una pagina.

È composto da un tag di apertura, uno di chiusura e del contenuto compreso tra i due. La differenza tra apertura e chiusura è data dallo slash che precede il nome del tag nella chiusura.

I TAG VANNO SEMPRE CHIUSI!

Esistono anche dei tag (pochissimi) che non hanno bisogno di una chiusura, in quanto autoconclusivi, infatti non contengono nulla al loro interno. Un esempio è il tag IMG.

I tag HTML possono avere degli attributi: informazioni aggiuntive che modificano il comportamento o il significato di un elemento. Gli attributi si inseriscono nel tag di apertura e sono sempre composti da un nome e un valore. Il tag <img>, per esempio, ha bisogno dell'attributo src per sapere quale immagine caricare, e dell'attributo alt per descriverla a chi non può vederla.

Due attributi particolarmente importanti — perché li useremo moltissimo con il CSS — sono class e id. Entrambi servono a identificare un elemento, ma con una differenza chiave: un id deve essere unico all'interno della pagina, mentre una class può essere assegnata a più elementi contemporaneamente.

Cos'è il CSS?

Cascading Style Sheets

Il CSS è un linguaggio separato che lavora mano nella mano con l'HTML e viene usato per dare uno stile alla pagina web, così che abbia un aspetto ancora migliore.

L'HTML crea la struttura dei contenuti che vogliamo far vedere sulla pagina, ma è il CSS a renderli piacevoli alla vista, agendo sui colori, la posizione degli elementi, aggiungendo effetti visivi, cambiando le dimensioni del font e così via.

Una regola CSS è composta da tre parti: un selettore, che indica a quale elemento HTML si vuole applicare lo stile; una proprietà, che indica cosa si vuole modificare (es. il colore, la dimensione del testo); e un valore, che indica come modificarlo. Insieme formano una dichiarazione.

Il CSS può essere scritto in tre modi: direttamente nell'attributo style di un tag HTML (inline), all'interno di un tag <style> nell'<head> del documento (interno), oppure in un file .css separato collegato tramite il tag <link> (esterno). Quest'ultimo è il metodo preferibile: mantiene il codice separato e riutilizzabile su più pagine.

Una delle proprietà CSS più importanti — e spesso fonte di confusione — è il box model. In CSS ogni elemento è trattato come una scatola rettangolare composta da quattro livelli: il contenuto, il padding (spazio interno tra il contenuto e il bordo), il border (il bordo vero e proprio) e il margin (spazio esterno tra l'elemento e gli altri elementi circostanti).

Il termine cascading nel nome CSS non è casuale: quando più regole si applicano allo stesso elemento, il browser segue un ordine di priorità ben preciso chiamato specificità. In linea generale, uno stile dichiarato inline ha priorità su quello in un file esterno, e un selettore per ID è più specifico di uno per classe, che a sua volta è più specifico di uno per tipo di elemento.
